# 日剧/电影字幕作为基准与测试用例来源

状态：想法，未实现。日期：2026-07-23。

## 起源

准备一集日剧或一部电影 + 字幕，把它当作"地面真值"来调参和评测。字幕同时提供三样东西：
- 语音边界（字幕时间戳 = 语音发生的时间）→ 调 VAD
- 转写文本（字幕文字）→ 评 ASR
- 场景结构（场景切换 / 说话人变化）→ 构造实时对话测试用例

## 想法 1（用户）：VAD 超参数网格搜索

用网格搜索扫 VAD 超参数（仅 VAD 阶段：进入/退出阈值、静音结束、最短语音、合并暂停阈值、合并上限），目标函数 = 预测片段时间轴与字幕时间轴的接近度。同一套音频 + 字幕也用来评 ASR（预测转写 vs 字幕）。

需要定义一个明确的对齐评分：
- 边界 F1：一个预测片段若与某条字幕重叠 > X% 则算匹配。
- 切分纯度：理想是「一条字幕 = 一个片段」。需分别惩罚过切分（一条字幕被切成多段）和欠切分（多条字幕被合成一段）——这两者对 ASR 的影响不同（过切分 → 短片段转写差；欠切分 → 一段里多句话）。
- 单一总分供网格搜索优化。

## 想法 2（用户）：从字幕场景构造 vieval 测试用例

字幕里的每个场景 = 一个场景（便利店、学校、职场…）。每场景抽对话轮次，生成 vieval 用例：
- 单轮：取一条「对方」台词，让教练出回复，按场景/水平判分。
- 多轮：取连续若干轮作为上下文，判连贯性。
- 变量：模型、是否开推理、推理深度、不同提示词模板。
- 目标：得出「某场景某水平下，哪个 模型×提示词×推理 配置教练得最好」。

## 补充想法

- **JLPT 难度自动打标**：用 JLPT 词汇表给每条字幕估 N5–N1，一部剧自动产出跨水平的用例，不用手标。
- **语义 oracle 而非 WER**：字幕是编辑过的，不等于精确转写。纯 WER 噪声大。改用 LLM-judge（vieval 风格）判 ASR 输出与字幕的语义匹配，而非字面 WER。单独评难点：重叠语、快语、填充词（あの/えーと）——教练场景里填充词要保留还是丢弃，值得专门评。
- **端到端延迟作为第二目标**：有音频+字幕时间轴，可测「语音结束（字幕 end）→ 教练产出候选（LLM done）」的真实延迟。网格搜索 VAD 时不仅看边界精度，还看 延迟 vs 精度 权衡（minSilenceDurationMs 越小触发越快但误切越多）。
- **说话人边界作为附加指标**：字幕的说话人切换 = 真值轮次边界。聚合器的「换人 → flush」可对照评测（即使没有演员声纹入库，边界检测部分仍可用字幕说话人标签当 oracle）。
- **合成学习者话术**：字幕只给「对方」（母语）侧。教练还需要「我」（学习者）侧。用 LLM 按各水平合成带相应错误的学习者话术，测教练纠错/建议回复的能力——这样端到端测教练，而不只是测「对方」侧。
- **退化鲁棒性**：剧里是干净棚录音频。加噪/混响/低音量再跑 VAD+ASR，看最优配置是否过拟合干净音频。
- **回归基准**：网格搜索 + vieval 选出最优 VAD 参数 + 最优 模型×提示词×推理 配置后，冻结一部分场景作回归基准。每次改 VAD/聚合器/提示词，重跑看分数不掉——把这部剧变成 CI 门禁。
- **实时回放对比**：有完整音频+字幕时间轴，可把音频按真实速度喂进 playground 管线（VAD→聚合→ASR→管线→LLM），把产出的实时时间轴与字幕时间轴可视化叠加（比如把字幕时间轴叠在 VAD 概率 sparkline 上）。既是 demo 也是验证工具。

## 待定
